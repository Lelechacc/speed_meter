1. 项目功能与硬件架构

1.1 主要功能

  • 基础交互：基于 LVGL 实现彩色 UI 界面。
  • 本地化支持：中英文切换 & 深浅模式切换。
  • 骑行数据：实时显示速度、电量、卫星数等。
  • 地图导航：支持局部 GPS 地图滚动与路书显示。

1.2 核心硬件选型

  • 主控：STM32F405 / STM32H750
  • 屏幕：240*320 TFT (ST7789) + 电容触摸
  • 传感器：ATGM336H (GPS) + LSM6DSM (陀螺仪)
  • 供电：3.7V 500mAh 三元锂电池

2. 核心技术难点与解决方案

难点一：地图动态铺设与性能瓶颈

实现地图无缝滚动需采用“九宫格”加载机制。在算力受限的情况下，单张 $256 \times 256$ 的图片在 $30\text{Hz}$ 刷新率下需要 $3\text{MB/s}$ 的数据吞吐。

解决方案： 在硬件端利用 DMA2D 硬件图形加速器搬运数据，并优化 SPI 传输效率，缓解 CPU 绘图压力。

难点二：数据绑定的线程安全

在定时器中断中直接修改 UI 绑定的数据会导致 HardFault。底层传感器更新与 GUI 渲染存在竞态关系。

解决方案： 引入 lv_async_call() 机制将修改动作委托给主循环，或使用 FreeRTOS 的互斥锁（Mutex）严格控制时序。
