⚙️ 项目功能与硬件架构

1.1 主要功能

  • 基础交互：基于 LVGL 实现彩色 UI 界面。
  • 本地化支持：中英文切换 & 深浅模式实时切换。
  • 骑行数据：实时显示速度、电量、卫星数等。
  • 地图导航：支持局部 GPS 地图滚动与路书显示。

1.2 核心硬件选型

  • 主控：STM32F405 / STM32H750
  • 屏幕：240*320 TFT (ST7789) + 电容触摸 (CST)
  • 传感器：ATGM336H (GPS) + LSM6DSM (陀螺仪)
  • 供电：3.7V 500mAh 三元锂电池

🔥 核心技术难点与解决方案

难点一：地图动态铺设与 SPI 性能瓶颈

实现地图无缝滚动需采用“九宫格”加载机制。在算力受限的情况下，单张 256x256 的图片在 30Hz 刷新率下需要 3MB/s 的数据吞吐量。F405 的 CPU 直接推屏会出现严重卡顿。

解决方案： 在硬件端考虑利用 DMA2D 硬件图形加速器 搬运数据，彻底解放 CPU 算力，实现地图画面的丝滑滑动。

难点二：数据绑定的线程安全问题

在定时器中断 (timer_handler) 中直接修改 UI 绑定的主题数据，会导致 LVGL 渲染线程冲突，极易引发单片机 HardFault 死机。

解决方案： 引入 lv_async_call() 将修改动作委托给主循环；或引入 FreeRTOS，使用互斥锁（Mutex）严格控制时序。
